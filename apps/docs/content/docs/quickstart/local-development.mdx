Use this page after the base Quickstart steps.
The goal is simple: make sure your local environment is healthy before you spend time modeling metadata or debugging permissions.

<DocCallout title="Expected outcome" tone="tip">
  You should finish this page with a healthy API, a working web login, and access to the product surface you need.
</DocCallout>

<DocSummary>
  <DocSummaryItem label="You need">
    A running local stack and a `.env` file based on `.env.example`.
  </DocSummaryItem>
  <DocSummaryItem label="You are done when">
    Health checks pass, login works, and you can open at least one product surface.
  </DocSummaryItem>
  <DocSummaryItem label="If it fails">
    Check role assignments first, then confirm the core auth and API URLs in `.env`.
  </DocSummaryItem>
</DocSummary>

## Start The Stack

```bash
pnpm infra:up
cp .env.example .env
pnpm dev
```

## Verify API Health

```bash
curl http://127.0.0.1:3001/health
```

The response should include `"status":"ok"` and `"ready":true`.

Default local ports:

- API: `http://localhost:3001`
- Web: `http://localhost:3000`
- Docs: `http://127.0.0.1:3002`
- Postgres: `127.0.0.1:5432`
- Redis: `127.0.0.1:6379`

## First Login

1. Open `http://localhost:3000`.
2. Use bootstrap or registration flow based on your `.env` settings.
3. Confirm you can access at least one of `/admin`, `/maker`, or `/worker`.

<Checklist>
  <li>If no surface is visible, check role assignments in Admin Center</li>
  <li>If login fails, verify `AUTH_BOOTSTRAP_TOKEN`, `SESSION_SECRET`, and `NEXT_PUBLIC_API_BASE_URL` in `.env`</li>
  <li>If passkeys fail, keep both the API and web app on `localhost` during local testing</li>
</Checklist>

## Optional Demo Seed

```bash
pnpm dev:seed
```

The seed targets `DEV_DEFAULT_TENANT_ID` when set, creates or updates the tenant `Northwind Industrial Group`, and adds sample entities, records, apps, forms, views, workflows, and sitemaps.

Seeded local users:

- Admin: `admin@qryvanta.local` / `admin`
- Standard user: `user@qryvanta.local` / `admin`

## Passkey Check

Use one hostname for both the web app and the API:

- `FRONTEND_URL=http://localhost:3000`
- `NEXT_PUBLIC_API_BASE_URL=http://localhost:3001`

Then:

1. Open `http://localhost:3000/login`.
2. Bootstrap a session or sign in with an existing local user.
3. Enroll a passkey.
4. Sign out and sign back in with the passkey flow.
