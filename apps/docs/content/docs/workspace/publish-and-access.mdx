Publish and access management should happen together.
This avoids runtime pages that users can reach but cannot use.

This page is useful for release windows and change approvals.

<DocSummary>
  <DocSummaryItem label="Use this page when">
    You are publishing metadata and need runtime access to stay correct on the same release.
  </DocSummaryItem>
  <DocSummaryItem label="Core rule">
    Publish changes, app bindings, and role access as one coordinated scope.
  </DocSummaryItem>
  <DocSummaryItem label="Validate with">
    A non-owner maker user and a non-owner worker user.
  </DocSummaryItem>
</DocSummary>

## Recommended Order

1. Update metadata and workflow drafts in Maker Center.
2. Run workspace publish checks.
3. Update app permissions and workflow permissions in Admin or Maker workflows.
4. Publish the selected entities, apps, and workflows together.
5. Validate in Worker Apps and workflow history with a non-owner user.
6. Record the change in your team change log.

## Validation Focus

- Entity and relation dependencies.
- Form and view reference integrity.
- Workflow trigger and action dependencies on published entities.
- Published-schema compatibility gates for breaking metadata changes.
- App role permission coverage for required actions.
- Workflow role coverage (`workflow.read`, `workflow.manage`) for release operators.
- Registration mode and invite policy after role changes.

## Pre-Release Check

1. Use a non-owner maker user for final publish checks.
2. Use a non-owner worker user for runtime validation.
3. Confirm workflow publish selections and versions are included in publish history.
4. Confirm audit events are recorded for publish and role changes.
5. Confirm rollback owner and rollback window are documented.

## Rollout Tip

Use role-scoped pilot users before broad tenant rollout.
This catches runtime permission gaps early.

## Rollback Guidance

If a rollout fails:

1. Remove access to impacted app routes for non-admin roles.
2. Disable or correct impacted workflows in Maker Center.
3. Revert metadata to last known good shape in Maker Center.
4. Publish the corrected entity/app/workflow scope.
5. Validate with pilot users before reopening access.
