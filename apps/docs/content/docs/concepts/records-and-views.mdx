Worker users handle records through forms and list views.
Those screens are defined in metadata and resolved inside app scope at runtime.

## Record Flow

1. Open app and entity route.
2. List records with the active view.
3. Open or create record with the active form.
4. Submit changes.
5. API validates values and permissions.
6. Updated data appears in list views based on current filter rules.

## What Controls the Experience

- Published form layout
- Published list view columns, sort, and filters
- App-scoped form and view defaults
- Role and runtime permissions
- Business rules that set values or block invalid writes

## Practical Reading Of The Runtime

When a runtime page looks wrong, break the problem into four checks:

1. Did the right metadata version get published?
2. Is the user in the right app and route?
3. Which form or view is the page resolving by default?
4. Which actions does the assigned role allow?

## Troubleshooting Tip

When users see different actions, compare their assigned roles and app permissions.

If users see old columns or old labels, check the latest published metadata version.
