Qryvanta is an open-source, self-hostable business platform.
You define entities, fields, views, apps, and workflows in metadata, then publish those changes into the runtime.

<DocCallout title="Status" tone="warn">
  Qryvanta is in active development and is not ready for production use yet.
  Use this docs site for local development, internal evaluation, and self-hosting preparation.
</DocCallout>

## What Exists Today

<Checklist>
  <li>Metadata entities and fields with draft and publish lifecycle</li>
  <li>Runtime record APIs generated from published metadata definitions</li>
  <li>Role-based product surfaces for admin, maker, and worker tasks</li>
  <li>Authentication with email/password, passkeys, MFA, and tenant switching</li>
  <li>Tenant-scoped RBAC, audit logging, security events, and PostgreSQL RLS coverage</li>
  <li>Workflow definitions with native schedules, webhooks, forms, inbound email, approvals, and queued execution</li>
</Checklist>

## Product Boundary

`qryvanta.org` is the OSS product and self-hosting path.

`qryvanta.com` is the planned managed-cloud option for teams that want hosted operations later.
Do not assume cloud-only behavior exists in this repository unless the docs say so.
