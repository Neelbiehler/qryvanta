Qryvanta is in active development and is not ready for production use yet.

Use this section to run Qryvanta in self-hosting environments.
It covers runtime setup, config controls, and operational checks.

If you run Qryvanta for internal teams, treat this section as required runbook material.

Use it before each release and during incidents.

## Use This Section For

- Runtime topology and deployment expectations.
- Environment variable and secret management.
- Transactional email transport for auth and invite flows.
- Log, metric, and trace baselines for troubleshooting and incident response.
- Worker queue operations in queued workflow mode.
