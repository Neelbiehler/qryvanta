This page is the route contract reference for documented API behavior.

## Global Route

- `GET /health` returns liveness and readiness with dependency status.

## Handler Rules

Handlers should stay transport-focused:

1. Parse request.
2. Call application service.
3. Map response.
4. Map errors to HTTP.

## Error Mapping

- `AppError::Validation` -> `400 Bad Request`
- `AppError::NotFound` -> `404 Not Found`
- `AppError::Conflict` -> `409 Conflict`
- `AppError::Forbidden` -> `403 Forbidden`
- `AppError::Internal` -> `500 Internal Server Error`
